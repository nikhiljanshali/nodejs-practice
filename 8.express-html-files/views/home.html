Node.js API Architecture Overview

A Professional Look at RESTful Service Structure

Introduction to Node.js API Structure

A well-structured Node.js API, typically leveraging the Express framework, adheres to architectural patterns that promote scalability, maintainability, and testability. The commonly adopted structure separates concerns into distinct layers: **Routes**, **Controllers**, **Services**, and **Models (Data Access Layer)**.

Core Components of the Structure

1. Routes (or Routers)

Routes define the endpoints (URIs) and map incoming HTTP requests (GET, POST, PUT, DELETE) to specific controller functions. They act as the entry point for all API traffic.

Example Responsibility: Defining /api/users.

// routes/userRoutes.js const express = require('express'); const router = express.Router(); const userController = require('../controllers/userController'); router.get('/', userController.getAllUsers); router.post('/', userController.createUser); module.exports = router;

2. Controllers

Controllers handle the request and response cycle. They validate input, call the appropriate service methods, and format the final response sent back to the client. They should contain minimal business logic.

Example Responsibility: Receiving a POST request and calling the service layer to create a user.

// controllers/userController.js const userService = require('../services/userService'); exports.createUser = async (req, res) => { try { const newUser = await userService.create(req.body); res.status(201).json({ message: 'User created successfully', data: newUser }); } catch (error) { res.status(400).json({ error: error.message }); } };

3. Services (Business Logic Layer)

Services contain the core business logic. They orchestrate data operations, perform complex calculations, and interact directly with the data access layer (Models). This layer ensures controllers remain lean.

Example Responsibility: Checking if a username already exists before saving to the database.

// services/userService.js const User = require('../models/userModel'); exports.create = async (userData) => { const existingUser = await User.findOne({ username: userData.username }); if (existingUser) { throw new Error('Username already taken'); } const user = new User(userData); return user.save(); };

4. Models (Data Access Layer / Persistence)

Models define the structure of the data (often using Mongoose schemas for MongoDB or Sequelize/Prisma for SQL) and handle direct interaction with the database (CRUD operations).

Example Responsibility: Defining the fields and validation rules for the User entity.

// models/userModel.js const mongoose = require('mongoose'); const UserSchema = new mongoose.Schema({ username: { type: String, required: true, unique: true }, email: { type: String, required: true, unique: true }, createdAt: { type: Date, default: Date.now } }); module.exports = mongoose.model('User', UserSchema);

Typical Flow of an API Request

The request follows a strict, unidirectional path to maintain separation of concerns:

  1. Client Request (e.g., Browser/App) sends HTTP request to the server.
  2. Routes Layer identifies the correct handler function based on the URL and HTTP method.
  3. Controller Layer extracts data from the request (body/params), executes validation, and delegates logic execution.
  4. Service Layer executes the required business rules, potentially involving multiple data operations.
  5. Model Layer interacts directly with the database (read/write).
  6. The response travels back up through the layers to the **Controller**, which formats the final JSON/XML response and sends it to the client.
